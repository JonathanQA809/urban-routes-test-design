# Urban Routes Test Design Project

## Overview

This project focuses on test design for the Urban Routes web application. The goal was to analyze functional requirements and create effective test cases using structured testing techniques.

## Scope

The project covered requirement analysis and test case design based on functional requirements FR-1 to FR-4.

## Key Activities

* Decomposed requirements into atomic blocks
* Identified equivalence classes for valid and invalid inputs
* Applied Boundary Value Analysis (BVA)
* Designed test cases based on EP and BVA techniques
* Executed test cases and recorded results

## Testing Techniques Used

* Equivalence Partitioning (EP)
* Boundary Value Analysis (BVA)

## Test Artifacts

* Requirement decomposition (atomic blocks)
* Equivalence classes and boundary values
* Test cases
* Test execution results

## Key Outcome

* Developed structured test cases based on formal test design techniques
* Improved ability to identify edge cases and invalid inputs
* Strengthened understanding of QA test design principles

## How to Use This Repository
